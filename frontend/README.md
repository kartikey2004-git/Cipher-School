# SQL Learn — Frontend

A modern, interactive SQL learning platform frontend built with Next.js. Provides a browser-based SQL editor, instant query execution, AI-powered hints, and assignment-based guided learning.

## 1. Project Overview

**SQL Learn Frontend** is a single-page application that connects to the SQL Learn backend API to deliver:

- **Assignment Browser** — Browse and select SQL practice assignments with difficulty indicators.
- **SQL Editor** — Monaco Editor-based SQL writing with syntax highlighting and autocomplete.
- **Instant Execution** — Run queries against a sandboxed PostgreSQL schema and see results in real time.
- **AI-Powered Hints** — Get syntax, logic, or approach hints generated by an LLM (or rule-based fallback).
- **Grading & Progress** — Submit queries for auto-grading and track completion across assignments.
- **Schema Viewer** — Inspect the tables and columns available in each assignment's sandbox.

## 4. Environment Variables

| Variable              | Required | Default                 | Description                                      |
| --------------------- | -------- | ----------------------- | ------------------------------------------------ |
| `NEXT_PUBLIC_API_URL` | No       | `http://localhost:5000` | Backend API base URL (used in client & rewrites) |

> Copy `.env.example` to `.env.local` and adjust:
>
> ```bash
> cp .env.example .env.local
> ```

The backend API URL is used in two places:

- **`next.config.ts`** — Configures Next.js rewrites to proxy `/api/*` requests to the backend.
- **`lib/api/client.ts`** — Used as the base URL for direct client-side `fetch` calls.

## 5. Installation & Setup

### Prerequisites

- **Node.js** ≥ 18
- **npm**, **yarn**, **pnpm**, or **bun**
- Backend API running (see [backend README](../backend/README.md))

### Steps

1. **Navigate to the frontend directory:**

   ```bash
   cd frontend
   ```

2. **Install dependencies:**

   ```bash
   npm install
   ```

3. **Set up environment variables:**

   ```bash
   cp .env.example .env.local
   ```

   Edit `.env.local` if your backend runs on a different port:

   ```env
   NEXT_PUBLIC_API_URL=http://localhost:5000
   ```

4. **Start the development server:**

   ```bash
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000) in your browser.

## 7. Technology Choices Explained

### Next.js 16 (App Router)

Chosen for its built-in server-side rendering, file-based routing, and seamless API rewrite proxying. The App Router provides React Server Components, layouts, and loading states out of the box. The `rewrites` in `next.config.ts` proxy all `/api/*` requests to the backend, avoiding CORS issues in development.

### Monaco Editor (`@monaco-editor/react`)

Powers the SQL editor — the same engine behind VS Code. Provides syntax highlighting, autocompletion, multi-cursor editing, and a familiar developer experience. Far superior to basic `<textarea>` inputs for writing SQL.

### TanStack React Query

Manages all server state (assignments, sandbox data, hints, grading results). Provides automatic caching, background refetching, loading/error states, and mutation handling without manual `useEffect` + `useState` boilerplate.

### Zustand

Lightweight state management for client-only state (editor content, UI panel visibility). Chosen over Redux for its minimal boilerplate and simplicity. Stores are just hooks — no providers, reducers, or action creators needed.

### Sass / SCSS Modules

Scoped styling per component using CSS Modules with SCSS syntax. Prevents class name collisions and keeps styles co-located with components. Global SCSS variables and themes are shared via `styles/globals.scss`.

### react-hot-toast

Lightweight toast notification library for success/error feedback after query execution, grading, and API errors.
